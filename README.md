# Enterprise Network Design & Implementation – Cisco Packet Tracer

## 📌 Project Overview
This project demonstrates the design and implementation of multiple network scenarios using Cisco Packet Tracer.  
The objective was to build scalable multi-router topologies and achieve full connectivity using static routing, while integrating essential network services such as DHCP, DNS, and secure wireless access.

The project consists of four practical scenarios (Q1–Q4), each increasing in complexity and configuration depth.

---

## 🧠 Scenario Breakdown

### 🔹 Q1 – Basic Static Routing
- Two LANs connected via two routers
- Inter-router network configuration
- Static route configuration
- End-to-end connectivity validation using ping

---

### 🔹 Q2 – Device Configuration & Basic Security Setup
This scenario focuses on router and switch configuration using Cisco IOS CLI.

Tasks implemented:
- Assigning hostnames to routers and switches
- Configuring interface IP addresses
- Setting enable secret password
- Configuring console line password
- Securing remote access
- Saving configurations to NVRAM
- Verifying configuration using:
  - show running-config
  - show ip interface brief

This part demonstrates hands-on experience with Cisco command-line configuration and basic network device hardening.

---

### 🔹 Q3 – Multi-Router Network with Services
- Three interconnected routers
- Multiple subnets
- DHCP server configuration
- DNS server configuration
- Web servers simulation
- Full static routing table configuration
- Connectivity verification between all networks

---

### 🔹 Q4 – Advanced Multi-Network Architecture
- Four routers interconnected
- Backbone network design
- Multiple LAN segments (192.168.20.0, 192.168.30.0, 192.168.40.0, 192.168.50.0)
- Wireless router configuration (WiFi 1 & WiFi 2)
- Static routing across all routers
- Cross-network connectivity validation
